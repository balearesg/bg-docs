<Title text="Manejo de proyectos." />

Los proyectos deben manejarse de la siguiente forma:

-   Manejar un branch master o main para el código puesto en producción.
-   El nombramiento de branchs debe realizarse por medio de la nomeclatura
    descriptiva siguiendo un estandar similar al de los commits convencionales:

Ejemplo:

"feature/", "fix/", "hotfix/",

Y se puede especificar

-   Branch de desarrollo para las integraciones en entorno de desarrollo.
-   Branch de calidad para las integraciones en entorno de calidad.
-   Cada desarrollador va a crear pull request cuando cierre un conjunto de tareas y se haya
    probado adecuadamente.
-   El codigo debera ser revisado en equipo o pares e integrado posteriormente a desarrollo.
-   En desarrollo se presentará al equipo de análisis quien deberá confirmar la entrega.
-   El coordinador del proyecto integrarà al entorno de calidad
-   Analisis deberà probar y confirmar.
-   El encargado integrara en produccion.
