## Types Esenciales

Cuando creamos componentes y necesitemos tipar la propiedad "children" del componente react tengamos en cuenta estos tipos :

- `React.Component`: Este tipo representa un componente de clase en React. Puedes utilizarlo como base para crear tus propios componentes de clase con TypeScript.

- `React.FC`: Este tipo representa un componente funcional en React. Es una forma más moderna de definir componentes y se usa con mayor frecuencia en proyectos de React recientes.

- `Props`: Define el tipo de las propiedades que se pasan a un componente. Por ejemplo, si tienes un componente llamado MyComponent que espera una prop llamada text, 
podrías definir el tipo como Props = `{ text: string }`.

`State`: Define el tipo del estado interno de un componente de clase. Puedes usarlo para garantizar que el estado tenga la forma correcta.

- `React.ReactNode`: Este tipo es útil para definir el tipo de contenido que puede estar dentro de un componente de React, como niños (children) en un componente.

- `Event Handlers`: Puedes definir tipos específicos para manejadores de eventos, como React.MouseEventHandler para manejar clics del mouse.

- `Ref Types`: Para trabajar con referencias a elementos DOM, puedes usar tipos como React.RefObject o React.MutableRefObject.

- `Context Types`: Si estás utilizando el contexto de React, puedes definir tipos para los valores de contexto y los objetos de contexto.
